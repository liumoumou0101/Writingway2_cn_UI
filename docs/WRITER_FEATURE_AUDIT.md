# 原生写作功能审计

最后更新：2026-06-29

这份文档记录当前原生写作页相对旧写作器已经实现、部分实现和未实现的功能。测试数据可随意重建，本轮重点是确认按钮和主流程是否真的可用。

## 2026-06-27 真实 API 回归结果

本轮使用用户数据目录中旧版 `writingway:aiSettings` 的 DeepSeek 配置做了真实 API 测试，测试过程中未打印或写入 API key。真实路径覆盖：

- 从写作页空状态点击“新建作品”，创建临时作品。
- 编辑初始场景并保存。
- 点击“新场景”，通过应用内命名弹窗创建场景。
- 点击“新章节”，通过应用内命名弹窗创建章节。
- 打开生成面板，填写 beat，预览 prompt。
- 使用 DeepSeek 真实 API 生成正文。
- 采纳生成结果，追加到当前场景。
- 保存场景，并通过项目 API 读回确认生成内容已经写入项目文件。

本轮真实 API 修复：

- 新增/重命名章节、场景不再依赖浏览器原生 `prompt`，改为应用内命名弹窗，避免桌面端点击后看起来无反馈。
- 写作页空状态增加“新建作品”和“返回书库”，新建作品入口现在可直接从写作页进入。
- 生成前会等待设置加载完成，避免初始化竞态导致生成拿到空 provider 配置。
- 生成配置改为使用后端返回的 `runtimeProvider`，避免 public settings 脱敏后把空 API key 传给生成器。
- DeepSeek 已加入新版 provider 白名单和设置页选项，并补上默认 endpoint。
- 新版 provider-stream 已迁移 DeepSeek V4 模型映射和 `thinking` 开关，非 thinking 模式会请求正文内容。
- Provider 连接测试不再把 4xx 状态误判为成功。
- AI 返回空正文时会作为错误处理，不再保存空生成历史。

本轮验证命令：

```powershell
npm run unit
npm run writer-audit
npm run desktop-mainline-test
```

另有一次临时 Playwright 真实 API 脚本通过：DeepSeek `deepseek-v4-flash` 返回正文，采纳后项目读回成功。该脚本只使用临时数据目录，测试结束后删除。

## 2026-06-28 写作体验修正

本轮针对笔记本屏幕和核心写作体验做了调整：

- 生成正文不再主要显示在右侧小面板，AI token 会直接流式写入中间正文编辑器。
- Prompt 预览保留，仍然通过“预览”按钮打开完整 prompt。
- 生成后可以“保留”“重试”“撤回”；撤回会恢复生成前正文。
- 辅助区支持右侧 / 下侧切换，并会记住本机偏好。
- 新增“改写”面板：选中正文后可润色、扩写、缩写、按人物语气改写，结果直接替换选区。
- 新增“人物”面板：可快速创建人物卡，并跳转到资料库编辑人物资料。

本轮验证：

```powershell
npm run writer-audit
npm run desktop-mainline-test
npm run unit
```

另用真实 DeepSeek 配置验证：生成内容会直接进入正文编辑器，撤回后正文恢复到生成前状态。

## 阶段 24 后的当前缺口判断

阶段 24 已经补齐此前列出的高价值写作页缺口：

- 场景标题内联编辑。
- 字号、行距、正文宽度、段落间距、字体、字数目标等写作偏好。
- 查找替换匹配数量、上一处/下一处导航、当前匹配定位和替换当前。
- 独立改写任务入口和常用改写预设。
- 生成任务类型：续写、beat 生成、场景摘要。
- 生成历史：当前场景过滤、复制、写入、重试、删除、任务标签、字数元信息和预览。
- 写作页导出 Markdown、TXT、HTML、EPUB、项目包。
- 导出选项：是否包含场景标题。
- 原生 TTS 语音选择和语速设置。

当前没有发现必须在人工测试前继续补齐的旧写作器核心功能。剩余项主要是体感打磨、发布验收和后置扩展。

## 2026-06-28 旧写作功能对应补齐

本轮目标不是重新设计 UI，而是先把旧写作器中对应的核心写作能力补到原生写作页，避免新写作模块只剩一个粗糙生成入口。

本轮已补齐：

- 正文生成继续直接流式写入中间正文编辑器，并保留 Prompt 预览、保留、重试、撤回。
- 改写预设扩展到旧写作器同级别：润色、压缩、扩写、少解释多呈现、感官、紧张感、快/慢节奏、对白、潜台词、情绪、角色声音、文学化、网文化、电影感、逻辑理清、上下文衔接、去套路、校对、换一种写法、自定义。
- 新增“重生成选区”：选中文段后按前后文重新生成替换内容，结果直接进入正文，并可保留/撤回。
- 新增特殊符号条：破折号、省略号、中文引号、书名式引号、全角空格等；`--` 会自动替换为中文破折号。
- 新增场景摘要和章节摘要 AI 生成，写入当前场景/章节元数据。
- 新增手动上下文面板：可选择资料条目、资料标签、章节摘要/全文、场景摘要/全文参与生成。
- 新增人物卡快捷入口：写作页可创建人物卡，并可跳转资料库继续编辑。
- 新增朗读/停止朗读，优先朗读选中文本，没有选区则朗读当前场景。
- 生成历史从只显示扩展为可复用、写入正文、删除。
- 写作页导出扩展为 Markdown、TXT、HTML、EPUB、项目包。
- 主导航、结构栏、辅助栏、专注模式、辅助栏右侧/下侧布局均可切换。

本轮审计新增覆盖：

- `npm run writer-audit` 已覆盖新建章节/场景、隐藏栏、专注、辅助布局、摘要生成、导出、搜索替换、特殊符号、朗读、正文生成、改写、选区重生成、人物卡、上下文选择、Prompt 管理、历史复用/写入/删除、保存、删除场景。

阶段 24 已补齐的后续项：

- 查找替换已有匹配数量、上一处/下一处导航、当前匹配定位和替换当前。
- 写作偏好已有字体、字号、行距、正文宽度、段落间距和字数目标。
- 导出已有 HTML/EPUB/项目包入口，并补充“导出包含场景标题”选项。
- 朗读设置已有原生语音选择和语速设置。
- 生成历史已有当前场景过滤、复制、写入、重试、删除、任务标签、字数元信息和结果预览。

仍然后置或不建议塞回写作页：

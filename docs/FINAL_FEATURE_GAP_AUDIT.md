# Writingway 最终功能缺口复核

最后更新：2026-06-29

本复核用于判断：在暂缓复杂半自动/自动小说工作流的前提下，原软件已有的高价值功能是否已经迁移到新的桌面原生主线，是否还存在发布前必须补齐的功能缺口。

## 结论

没有发现新的“必须先补功能再进入人工测试”的阻断项。

当前状态可以进入发布前人工验收。剩余工作主要是：

- 人工验证打包产物、installer、portable 和真实用户路径。
- 视觉/交互体感打磨，尤其是写作页密度和设置页排布。
- 复杂半自动/自动小说工作流后续专项设计。
- 远端/云备份类扩展后置。

## 已补齐的原软件核心能力

### 项目和本地数据

- 新建、打开、编辑、保存项目。
- 项目目录格式作为新主线存储。
- 旧 JSON snapshot 导入。
- Writingway 1 文件夹导入。
- 项目包导入和导出。
- 书库元数据编辑、移出书库、定位项目目录。

### 写作页

- 原生章节/场景管理：新增、重命名、删除、排序。
- 正文编辑、保存、自动保存、未保存提示。
- 场景标题内联编辑。
- 字数统计、字数目标。
- 字号、行距、正文宽度、段落间距、字体偏好。
- 主导航/结构栏/辅助栏隐藏。
- 辅助栏右侧/底部切换。
- 专注模式。
- 特殊符号插入和 `--` 自动替换为破折号。
- 朗读/停止朗读。
- 原生 TTS 语音和语速设置。

### 查找替换

- 当前场景搜索。
- 匹配数量。
- 上一个/下一个匹配导航。
- 当前匹配定位。
- 替换当前。
- 全场景替换。

### 生成、改写、Prompt、上下文

- 原生 Provider 设置。
- DeepSeek/OpenAI-compatible 等 provider 配置。
- 生成正文、按 beat 生成、场景摘要。
- Prompt 预览。
- 生成取消、保留、重试、撤回。
- 追加末尾、插入光标、替换选区。
- 改写预设：润色、扩写、精炼、风格改写等。
- 选区重生成。
- Prompt 模板管理。
- 手动上下文选择：资料条目、资料标签、章节、场景。
- 上下文摘要显示当前生成将带入的内容。

### 生成历史

- 历史展示。
- 当前场景过滤。
- 结果预览。
- 任务标签、字数元信息。
- 复用提示。
- 复制结果。
- 写入正文。
- 重试并发起新生成。
- 删除记录。

### 资料库、Workshop、阅读器

- 原生资料库列表、搜索、类型筛选、编辑、删除。
- 资料条目标签、别名、摘要、正文和默认加入上下文。
- 原生 Workshop 会话。
- Workshop 支持引用资料/场景上下文。
- Workshop 输出可转资料、摘要或正文片段。
- 原生阅读器显示项目正文。
- 阅读器基础排版设置。

### 备份、恢复、导出

- 原生恢复中心。
- 备份列表、搜索、筛选、预览。
- 单场景恢复。
- 整项目恢复。
- 恢复为新项目。
- 恢复前自动创建快照。
- Markdown、TXT、HTML、EPUB 导出。
- 项目包导出。
- 导出选项：是否包含场景标题。

## 不作为当前阻断项的旧能力

这些能力不是当前发布前必须迁移项：

- GitHub/Gist 远端备份：保留旧兼容实现，未来作为独立扩展重写。
- OneDrive / Google Drive：仍按占位或未来扩展处理，不纳入当前验收。
- 旧 Web UI 内部调试面板、临时面板、多套滑出面板：不照搬。
- 旧 iframe 写作器：保留为显式“兼容写作器”入口，不再作为主流程。
- 复杂半自动/自动小说工作流：目前只验收最小人工确认闭环，复杂编排、批量生成、分支回滚后置。

## 仍需人工验收的发布项

进入人工测试前不需要再补大功能，但需要人工确认：

- `release/win-unpacked/Writingway.exe` 可以启动。
- portable exe 可以启动，且不和开发服务器端口冲突。
- installer 可以安装并启动。
- 真实项目持久化：新建项目 -> 写正文 -> 关闭 -> 重开 -> 正文仍存在。
- 真实备份恢复：创建备份 -> 预览 -> 恢复为新项目。
- 导入旧 JSON、项目包、Writingway 1 文件夹的真实数据路径。
- 使用真实 API key 做一次原生生成、改写、摘要生成。
- 打开兼容写作器不会影响原生主线数据。
- 笔记本屏幕下写作页视觉密度、按钮可点击性、文本不重叠。

## 建议下一步

1. 更新用户验收清单，使它反映阶段 24 后的原生写作页能力。
2. 运行发布前自动验证：`npm test`、`npm run backup-test`、`npm run pack`、`npm run packaged-smoke`。
3. 进行人工验收。
4. 人工验收通过后，再考虑复杂半自动/自动小说工作流专项。

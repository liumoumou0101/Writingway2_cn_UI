# Writingway 最终功能缺口复核

最后更新：2026-06-29

本复核用于判断：在暂缓复杂半自动/自动小说工作流的前提下，原软件已有的高价值功能是否已经迁移到新的桌面原生主线，是否还存在发布前必须补齐的功能缺口。

## 结论

没有发现新的“必须先补大功能再进入人工测试”的阻断项。

但 2026-06-29 对“我们在旧开源项目上后续增加的增强功能”重新复核后，需要修正一个判断：此前本文件按“是否有入口/是否能跑通”判断为基本补齐，粒度偏粗。旧增强版的“改写/选区重生成”在新版原生写作页里虽然有入口、预设和可运行路径，但交互完整度、可控性和安心感最初没有达到旧增强版效果。阶段 25 已补上原文预览、保存的 rewrite Prompt、预设说明、结果预览后接受和上下文开关。

当前状态可以进入发布前人工验收。剩余工作主要是：

- 人工验证打包产物、installer、portable 和真实用户路径。
- 视觉/交互体感打磨，尤其是写作页密度和设置页排布。
- 人工验证原生改写工作台：确认右侧面板中的原文/结果对照、保存的改写 Prompt、结果接受流程在真实屏幕上足够顺手。
- 复杂半自动/自动小说工作流后续专项设计。
- 远端/云备份类扩展后置。

- 项目包导入和导出。
- 书库元数据编辑、移出书库、定位项目目录。

- 正文编辑、保存、自动保存、未保存提示。
- 场景标题内联编辑。
- 字数统计、字数目标。
- 字号、行距、正文宽度、段落间距、字体偏好。
- 主导航/结构栏/辅助栏隐藏。
- 辅助栏右侧/底部切换。
- 专注模式。
- 特殊符号插入和 `--` 自动替换为破折号。

### 查找替换

- 当前场景搜索。
- 匹配数量。
- 上一个/下一个匹配导航。
- 当前匹配定位。
- 替换当前。
- 全场景替换。

### 生成、改写、Prompt、上下文

- 原生 Provider 设置。
- DeepSeek/OpenAI-compatible 等 provider 配置。
- 生成正文、按 beat 生成、场景摘要。
- Prompt 预览。
- 生成取消、保留、重试、撤回。
- 追加末尾、插入光标、替换选区。
- 改写预设：润色、扩写、精炼、风格改写等。

- 手动上下文选择：资料条目、资料标签、章节、场景。
- 上下文摘要显示当前生成将带入的内容。

### 后续增强功能复核修正：改写未完全等效

旧增强版改写能力包括：

- 选中文本后打开独立改写弹窗。
- 左侧保留原文预览，右侧选择内置预设或保存的 rewrite Prompt。
- 支持编辑改写要求。
- 支持预览最终 Prompt。
- AI 输出先进入结果框，用户确认后再替换正文。
- 支持接受、重试、放弃。
- 选区重生成弹窗支持编辑原选区、填写重生成要求、选择是否使用前后文上下文。
- 选区重生成默认带入前后各约 8000 字符上下文，并在接受前校验原文未变化。

新版原生写作页当前状态：

- 已有：内置改写预设、选中文本改写、选区重生成、Prompt 预览、撤回/重试、历史记录。
- 已有但体验不同：结果会直接流式写入正文编辑器，通过“保留/撤回”确认；这比旧版更快，但缺少独立结果预览和原文/结果对照。
- 不足：改写面板没有“保存的改写 Prompt”选择。
- 不足：普通改写只带前后约 1200 字符场景上下文，旧版选区重生成是前后各约 8000 字符，且可开关上下文。
- 不足：没有独立的改写结果工作台，用户很难在接受前比较原文和结果。
- 不足：没有把旧版 rewrite preset 的说明文案展示出来，只提供下拉选项。

阶段 25 处理后：

- 已补强：原文预览。
- 已补强：保存的 rewrite Prompt 选择。
- 已补强：preset description。
- 已补强：结果先预览，点击“保留”后才替换正文。
- 已补强：接受前校验原选区是否变化。
- 已补强：选区重生成上下文开关，保留前后各约 8000 字符上下文能力。
- 已补强测试：`writer-audit` 覆盖保存 Prompt、预设覆盖、接受前正文不变、上下文关闭影响 Prompt。

结论：改写能力已重新达到旧增强版的关键控制和安全能力。剩余风险主要是 UI 体感：它现在仍嵌在右侧辅助面板中，是否足够宽敞需要人工测试判断。

## 不作为当前阻断项的旧能力

这些能力不是当前发布前必须迁移项：

- GitHub/Gist 远端备份：保留旧兼容实现，未来作为独立扩展重写。
- OneDrive / Google Drive：仍按占位或未来扩展处理，不纳入当前验收。
- 旧 Web UI 内部调试面板、临时面板、多套滑出面板：不照搬。
- 旧 iframe 写作器：保留为显式“兼容写作器”入口，不再作为主流程。
- 复杂半自动/自动小说工作流：目前只验收最小人工确认闭环，复杂编排、批量生成、分支回滚后置。

## 仍需人工验收的发布项

进入人工测试前不需要再补大功能，但需要人工确认：

- `release/win-unpacked/Writingway.exe` 可以启动。
- portable exe 可以启动，且不和开发服务器端口冲突。
- installer 可以安装并启动。
- 真实项目持久化：新建项目 -> 写正文 -> 关闭 -> 重开 -> 正文仍存在。
- 真实备份恢复：创建备份 -> 预览 -> 恢复为新项目。
- 导入旧 JSON、项目包、Writingway 1 文件夹的真实数据路径。
- 使用真实 API key 做一次原生生成、改写、摘要生成。
- 打开兼容写作器不会影响原生主线数据。
- 笔记本屏幕下写作页视觉密度、按钮可点击性、文本不重叠。

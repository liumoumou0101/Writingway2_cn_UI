# 桌面 UI 优化待办清单

## 阶段 37：非写作 / 非阅读模块整体 UI 产品化（进行中）

用户复核结论：当前软件已经可用，但除了反复打磨过的“写作”和“阅读”，其他模块整体仍然别扭。主要问题不是单个按钮或间距，而是页面组织、信息层级、模块气质和跨模块操作语言不统一。

阶段目标：把书库、资料库、Workshop、Workflow、恢复中心、设置页从“功能堆叠页”推进到“成熟桌面创作软件工作台”。写作页和阅读页本轮只做兼容检查，不作为主要改造对象。

### 总体问题判断

- [ ] 书库：第一版已经从后台列表改成作品工作台，但项目卡、左侧工作台、导入维护区还可以更像“作品管理”而不是“文件管理器”。
- [ ] 资料库：功能强，但 UI 仍偏表单堆叠；人物卡、资料卡、注入规则、触发条件需要更清晰的卡片化编辑体验。
- [ ] Workshop：对话功能可用，但视觉上还不像创作讨论空间；消息、引用、输出转资料 / 摘要 / 正文的转化流程需要重排。
- [ ] Workflow：当前更像流程调试面板；需要把步骤、状态、产物、用户确认做成“创作流程看板”。
- [ ] 恢复中心：可用但缺少安全感；备份列表、预览、差异、恢复动作需要更明确地表达“不会误操作”。
- [ ] 设置页：功能开始变多后仍然像大表单；Provider、模型、API 配置组、路径、朗读、生成参数需要分区、摘要和状态反馈。
- [ ] 全局导航：左侧导航能用，但模块之间“我现在在哪个项目 / 哪个场景 / 哪个上下文”的连续性仍可加强。
- [ ] 空状态：很多页面的空状态还只是提示文字，需要变成可执行入口，并和当前项目状态联动。
- [ ] 下拉框 / 表单控件：视觉基线已经统一过，但复杂配置表单仍需要更清楚的分组、解释和禁用态。
- [ ] 2K 与笔记本适配：目前做了基础无横向溢出检查，后续要检查内容密度、行长、卡片列数、侧栏宽度和底部操作区。

### 设计原则

- [ ] 不再每个模块单独“补皮肤”，先建立统一模块框架：页面标题区、当前项目区、主操作区、列表区、详情区、空状态区、危险操作区。
- [ ] 控制卡片数量和嵌套，不把所有内容都装进卡片；卡片只用于重复项目、详情编辑器、对话消息、流程步骤。
- [ ] 让主操作永远清楚：书库是继续写作 / 新建 / 导入；资料库是新建资料 / 编辑卡片 / 注入规则；Workshop 是发送问题 / 转化输出；Workflow 是启动 / 审核 / 应用产物；恢复是预览 / 恢复。
- [ ] 把危险操作从常用操作中分离，尤其是移出书库、删除资料、删除对话、恢复覆盖。
- [ ] 每个模块都要有“无项目、无数据、已选择、加载中、失败、禁用”的完整状态。
- [ ] 页面密度要适合长期使用，避免营销式大块文案，也避免工程后台式密集按钮。

## 总体原则

- 保留当前暗色桌面工作台气质，但减少“工程后台 / 调试面板”的感觉。
- 优先统一控件和信息层级，再做大模块重排。
- 卡片圆角保持克制，重复卡片不超过 8px。
- 不做营销式首页，不做大面积装饰背景。
- 操作工具要紧凑、可扫描、适合长期写作。
- 所有 UI 优化必须保留现有功能和数据行为。

# 用户验收清单

最后更新：2026-06-29

这份清单用于你手动测试当前重构后的桌面版。建议优先测试打包产物，而不是开发模式。

## 推荐启动方式

优先启动：

```text
release/win-unpacked/Writingway.exe
```

也可以测试 portable：

```text
release/Writingway 1.0.0.exe
```

installer 测试：

```text
release/Writingway Setup 1.0.0.exe
```

## 第一轮冒烟测试

- [ ] 应用可以打开，默认进入桌面书库，而不是旧 `main.html` 页面。
- [ ] 顶部/侧边导航可以切换：书库、写作、阅读、资料、讨论、工作流、恢复、设置。
- [ ] 主界面功能导航栏可以隐藏和显示。
- [ ] 打开项目时不会自动加载旧兼容 iframe。
- [ ] 点击“兼容写作器”时，旧 writer 才会按需加载。

## 项目和编辑器

- [ ] 新建项目。
- [ ] 新建或选择章节/场景。
- [ ] 场景标题可以内联编辑。
- [ ] 写作页左侧结构栏可以隐藏和显示。
- [ ] 写作页右侧辅助栏可以隐藏和显示。
- [ ] 专注模式可以进入和退出。
- [ ] 字号、行距、正文宽度、段落间距、字体和字数目标可调整。
- [ ] 编辑正文并保存。
- [ ] 关闭应用后重新打开，项目仍在书库。
- [ ] 重新打开项目后，正文仍存在。
- [ ] 修改项目名称、描述、状态、标签后，书库显示正确。

## 阅读器

- [ ] 从书库或写作视图进入阅读器。
- [ ] 阅读器能显示当前项目正文。
- [ ] 编辑器保存后，阅读器刷新内容正确。
- [ ] 阅读器目录、翻页/滚动和基础排版没有明显错位。

## 资料库和上下文

- [ ] 新增资料条目。
- [ ] 编辑资料条目的类型、标签、摘要、正文。
- [ ] 搜索和类型筛选可用。
- [ ] 在生成或讨论中引用资料时，上下文能被带入。

## 原生生成和设置

- [ ] 在设置页配置 API provider、endpoint、model、temperature、max tokens。
- [ ] 在设置页选择 TTS 语音和语速。
- [ ] 原生生成面板能读取这些配置。
- [ ] 生成结果可以采纳到当前场景。
- [ ] 生成结果可以追加末尾、插入光标处、替换选中文本。
- [ ] 改写预设仍可用。
- [ ] 生成历史可以复制、写入、重试、删除，并按当前场景过滤。
- [ ] 上下文面板能显示当前生成会带入哪些资料、标签、章节和场景。

## Workshop / 创作讨论

- [ ] 新建讨论会话。
- [ ] 发送消息并获得回复。
- [ ] 通过 `@[资料]` 或 `#[场景]` 引用上下文。
- [ ] 将讨论输出保存为资料、摘要或正文片段。

## 备份和恢复

- [ ] 创建一次手动备份。
- [ ] 恢复中心能看到该备份。
- [ ] 备份预览可用。
- [ ] 恢复为新项目可用。
- [ ] 场景级恢复可用。
- [ ] 旧备份目录中的备份仍能被读取。

## 导入导出

- [ ] 导出 Markdown。
- [ ] 导出 TXT。
- [ ] 导出 HTML。
- [ ] 导出 EPUB。
- [ ] 切换“导出包含场景标题”选项后，导出内容符合预期。
- [ ] 导出项目包。
- [ ] 导入项目包。
- [ ] 导入旧 JSON snapshot。
- [ ] 如有 Writingway 1 数据，测试导入路径。

## 半自动工作流

当前只验收最小人工确认闭环；复杂自动化、批量连续生成、分支回滚等后置。

- [ ] 打开工作流视图。
- [ ] 输入项目设定并启动工作流。
- [ ] 生成章节大纲。
- [ ] 批准大纲进入场景草稿。
- [ ] 生成场景草稿。
- [ ] 点击“采纳草稿”，确认正文写入项目但 workflow 未结束。
- [ ] 最终批准后，workflow 标记为完成。
- [ ] 测试“退回步骤”和“取消运行”。

## 发布产物专项

- [ ] `release/win-unpacked/Writingway.exe` 可启动。
- [ ] `release/Writingway 1.0.0.exe` portable 可启动。
- [ ] `release/Writingway Setup 1.0.0.exe` installer 可安装并启动。
- [ ] 没有开发服务器占用端口时，产物能自行启动本地服务。
- [ ] 如端口 8000/8001 被占用，错误提示可理解。

## 已知边界

- 复杂小说自动/半自动工作流暂不作为本轮功能补齐目标，只验收最小真实闭环。
- `main.html` 仍保留为兼容写作器和旧 UI 测试目标。
- `asar` 仍关闭，打包产物使用普通目录资源。
- OneDrive 和 Google Drive 仍是占位，不要按已实现功能验收。
- 原生写作页功能对照详见 `docs/WRITER_FEATURE_AUDIT.md`。

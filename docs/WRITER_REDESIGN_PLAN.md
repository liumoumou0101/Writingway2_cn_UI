# 原生写作页重设计计划

最后更新：2026-06-27

这份计划用于修正当前原生写作页“功能迁移完成，但体验太粗糙”的问题。写作页是 Writingway 的核心，不应该只是旧功能的临时替代品。

## 当前判断

现在的原生写作页已经具备基础能力：

- 项目、章节、场景打开和保存。
- 章节/场景新建、重命名、删除、排序。
- 场景正文编辑。
- 场景摘要、标签、POV、时态。
- 搜索和替换。
- Markdown/TXT 导出。
- 原生生成、Prompt 预览、接受、重试、丢弃。
- 提示词管理和生成历史。
- 兼容写作器按需打开。

但它仍然不像一个真正的写作工作台，主要问题是：

- 工具条堆叠过多，常用动作、管理动作、高风险动作混在一起。
- 正文编辑区域被元数据和生成面板挤压，缺少专注写作感。
- 生成面板占用底部固定空间，导致普通写作时也像在操作后台工具。
- 场景树信息密度低，但缺少真正有用的章节/场景状态。
- 元数据编辑太显眼，实际写作时摘要/标签/POV 应该是可收起的辅助面板。
- 查找替换暴露在左栏顶部，干扰项目结构导航。
- 提示词管理出现在写作流里太重，应成为设置/Prompt 管理功能，写作页只保留选择和预览。
- 缺少写作者常用的编辑器能力：撤销友好保存、自动保存状态、字数目标、当前段落/选区生成、插入位置选择、版本感知。
- 旧写作器里有价值的生成/改写能力没有被完整地重新组织进原生写作页。

## 设计目标

写作页应该围绕三个状态设计：

1. **写正文**：最常用，正文区最大，界面安静。
2. **组织结构**：管理章节、场景、顺序、摘要和状态。
3. **AI 辅助**：按当前场景、选中文本、beat、资料上下文生成或改写。

不是所有功能都应该同屏常驻。常驻的应该只有：

- 项目/章节/场景定位。
- 当前场景标题。
- 保存状态。
- 正文编辑器。
- 快速生成/改写入口。

其他功能应该进入可折叠侧栏、菜单、抽屉或专门面板。

## 必须保留并做好的旧功能

这些能力不能因为“拆到资料库/工作流”就从写作页消失：

- 章节和场景管理：新建、重命名、删除、移动、拖拽排序。
- 场景摘要、标签、POV、时态。
- 正文生成：根据当前场景、beat、前文、资料库上下文生成。
- 续写/插入：生成结果应能插入光标处、追加到末尾或替换选区。
- 改写：对选中文本进行润色、扩写、缩写、改变语气、保持风格改写。
- Prompt 预览：生成前能查看最终请求。
- 生成历史：可回看、重用、再次插入。
- 查找替换。
- 导出 Markdown/TXT。
- 兼容写作器：保留为对照和兜底，但不作为主流程。

这些能力可以改变入口和布局，但不应该消失。

## 建议的新布局

### 左侧：项目结构栏

用于章节/场景导航，保持稳定：

- 顶部显示项目名、总字数、章节数/场景数。
- 搜索入口改成图标或折叠搜索，不常驻两个输入框。
- 章节可以折叠。
- 场景卡显示：标题、字数、状态/标签小标记、摘要提示。
- 章节/场景的管理动作放在右键菜单或更多菜单中。
- 支持拖拽排序，保留上移/下移作为键盘友好 fallback。

### 中央：正文编辑器

这是主角：

- 更大的编辑空间，默认隐藏元数据栏和生成历史。
- 顶部只显示当前章节、场景标题、保存状态、字数。
- 场景标题应可直接点击编辑，而不是只通过“重命名”按钮。
- 正文编辑器增加舒适宽度选项：普通、宽屏、专注。
- 增加专注模式：隐藏左侧栏和右侧辅助栏，只留正文和极少状态。
- 保存状态清晰区分：未保存、保存中、已保存、自动保存失败。

### 右侧：辅助面板

右侧用 tab 或分段控制切换，不要全部展开：

- `生成`：beat、模板、上下文提示、生成按钮、结果操作。
- `改写`：选中文本改写、预设、结果操作。
- `资料`：当前场景相关资料、可快速插入引用。
- `元数据`：摘要、标签、POV、时态、场景状态。
- `历史`：生成历史、可插入/复制/重试。

默认打开 `生成` 或记住用户上次选择。

## AI 生成设计

写作页里的 AI 不应该只是“底部一个大输入框”。建议拆成明确任务：

- `续写当前场景`
- `根据 beat 生成下一段`
- `扩写选中文本`
- `润色选中文本`
- `缩写选中文本`
- `改成更紧张/更抒情/更口语`
- `生成场景摘要`
- `根据资料补充细节`

结果操作：

- 插入光标处。
- 替换选区。
- 追加到场景末尾。
- 保存到生成历史。
- 发送到 Workshop 继续讨论。
- 保存为资料条目。

这样旧功能里真正有价值的“生成/改写/上下文”都保留，但入口会更清楚。

## 和资料库、Workshop、工作流的边界

写作页不是把一切都做进去。

### 写作页负责

- 当前正文和当前场景。
- 与当前场景直接相关的生成、改写、摘要、插入。
- 快速查看/引用资料。

### 资料库负责

- 角色、地点、组织、设定、时间线等长期知识。
- 资料条目的完整编辑。
- 跨场景引用和知识维护。

### Workshop 负责

- 开放式讨论、头脑风暴、结构问题、人物动机分析。
- 将讨论输出转为资料、摘要、草稿。

### 工作流负责

- 多步骤、跨章节、需要人工确认的半自动流程。
- 大纲、连续场景草稿、批量生成、回滚、事件日志。

写作页可以调用这些能力，但不应该承载复杂状态机。

## 分阶段实施

### 阶段 W1：信息架构和布局重做

- 三栏布局：结构栏 / 正文区 / 辅助面板。
- 移除顶部堆叠工具条，改为分组菜单和局部动作。
- 元数据从左侧常驻改为右侧 `元数据` tab。
- 生成从底部常驻改为右侧 `生成` tab。
- 保留所有现有功能，不做大范围业务逻辑重写。

验收标准：

- 普通写作时正文区域显著变大。
- 常用动作不需要在一堆按钮里找。
- 章节/场景管理仍完整可用。

### 阶段 W2：编辑器体验补强

- 场景标题可直接编辑。
- 自动保存状态更清楚。
- 增加专注模式。
- 增加编辑器宽度/字号/行距偏好。
- 增加字数目标和当前场景进度。
- 查找替换改为弹出/抽屉式工具。

验收标准：

- 可以长时间写正文而不被工具 UI 干扰。
- 保存状态不会让用户困惑。

### 阶段 W3：AI 生成和改写重做

- 增加任务选择：续写、beat 生成、扩写、润色、缩写、风格改写、摘要。
- 支持选区生成/改写。
- 结果支持插入光标、替换选区、追加末尾。
- 生成历史支持重用、复制、插入。
- Prompt 预览保留，但不打断主流程。

验收标准：

- 用户能明确知道自己在让 AI 做什么。
- 生成结果能自然回到正文，而不是只能“接受追加”。

### 阶段 W4：上下文和资料联动

- 右侧 `资料` tab 显示当前场景相关资料。
- 支持一键加入/排除上下文。
- 支持从正文选中文本创建资料条目。
- 支持场景摘要自动更新或辅助生成。

验收标准：

- 资料库不再像独立页面，能在写作时真正帮到当前场景。

### 阶段 W5：旧功能对照和兜底清理

- 对照旧写作器仍独有的高价值能力。
- 能迁移的迁移，没必要的明确归档。
- `兼容写作器` 保留，但视觉上弱化为兜底入口。
- 更新验收清单和 desktop-mainline 测试。

验收标准：

- 日常写作不需要打开旧 iframe。
- 旧 iframe 只用于对照、迁移和极少数兜底场景。

## 测试策略

需要新增或更新：

- desktop writer layout smoke：确认三栏、tab、正文区、辅助面板渲染。
- editor persistence：标题、正文、摘要、标签、POV、时态保存并重开。
- scene operations：章节/场景新建、重命名、删除、排序。
- generation insertion：追加、插入光标、替换选区。
- rewrite selection：选中文本改写后可替换。
- focus mode：切换后布局不破、不丢状态。
- packaged smoke：至少覆盖新写作页启动和保存主路径。

## 不应该做的事

- 不应该继续在当前顶部工具条里追加按钮。
- 不应该把复杂工作流提前塞进写作页。
- 不应该把资料库完整编辑器搬进写作页。
- 不应该为了复刻旧 UI 而保留旧 Web 时代的混乱布局。
- 不应该只做视觉皮肤，必须同时整理功能入口。

## 推荐下一步

先做阶段 W1 和 W2。

原因：这两步不需要重写 generation core 或 workflow，但能立刻把核心写作体验从“临时工具页”提升为“可长期使用的写作工作台”。等布局和编辑器体验稳定后，再做 W3 的生成/改写体验重构。

## 第一版实现记录

2026-06-27 已完成第一版写作页重排：

- 写作页改为三栏：左侧项目结构 / 中央正文编辑器 / 右侧辅助面板。
- 左侧结构栏保留章节、场景、新章节、新场景和拖拽排序。
- 中央正文区成为主区域，顶部只保留场景定位、保存状态、字数、专注模式和保存按钮。
- 右侧辅助面板拆为 `生成`、`元数据`、`结构`、`查找`、`历史`。
- 元数据从正文左侧常驻移入右侧 `元数据` 面板。
- 查找替换从左栏常驻移入右侧 `查找` 面板。
- 章节/场景高风险管理动作移入右侧 `结构` 面板。
- 生成从底部常驻改为右侧 `生成` 面板。
- 新增专注模式，隐藏左右栏，只保留正文编辑器。
- 新增主导航隐藏/显示。
- 新增结构栏隐藏/显示。
- 新增辅助栏隐藏/显示。
- 生成结果新增采纳方式：追加到场景末尾、插入光标处、替换选中文本。
- 更新 `tests/desktop-library.js`，覆盖新面板切换、专注模式和生成采纳路径。
- 新增 `tests/writer-button-audit.js` 和 `npm run writer-audit`，覆盖写作页按钮审计。

这版仍然不是最终写作体验。下一版建议继续做：

- 场景标题内联编辑。
- 编辑器字号、行距、正文宽度偏好。
- 查找替换抽屉的交互细化。
- 生成任务拆分：续写、扩写、润色、缩写、风格改写、摘要。
- 生成历史支持一键插入、复制、重试。

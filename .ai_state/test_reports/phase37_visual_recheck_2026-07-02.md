## 关键问题

### 书库

截图：

- `bookshelf-small-laptop-1280x720.png`
- `bookshelf-laptop-1366x768.png`

问题：

- 左侧工作台高度过高，1280x720 下“维护”区只露出标题，按钮被截断。
- 保存路径文本过长，虽然没有造成横向溢出，但视觉上显得硬、挤、偏工程。
- 右侧空状态可执行了，但整体仍显得像“左边控制台 + 右边空框”，作品感还不够。

建议：

- 左侧工作台改为更紧凑结构，导入 / 维护可折叠或收进次级操作。
- 保存路径做单行省略，必要时 hover / tooltip 展示完整路径。
- 无项目状态右侧增加更明确的导入入口分组，减少左侧按钮压力。

### 资料库

截图：

- `compendium-small-laptop-1280x720.png`

问题：

- 资料库空状态已比之前好，disabled 表单问题已修复。
- 未打开项目时顶部 context strip 占用 56px，信息价值不高。
- 左栏仍偏“筛选表单 + 大空白”，还没像上下文资料管理器。

建议：

- 未打开项目时 context strip 应更紧凑，或由页面内空状态承担引导。
- 资料库左栏需要“资料类型 / 注入状态 / 最近编辑”等产品化分组，而不是纯表单。
- 后续有项目状态下再重点检查人物卡编辑体验。

### Workshop

截图：

- `workshop-small-laptop-1280x720.png`

问题：

- 未打开项目时仍显示完整聊天工作区和输入框，像可以操作但实际不可用。
- 输出转化动作没有在空状态中表达价值。
- 侧栏和主区缺少“创作讨论空间”的层级。

建议：

- 无项目时隐藏聊天输入区，展示“打开项目后开始讨论”的可执行空状态。
- 无对话时主区展示创建第一场讨论，而不是保留空消息框。
- 37C 应优先处理 Workshop。

建议：

- 37C 继续按“创作流程看板”重排。

### 恢复中心

截图：

- `recovery-small-laptop-1280x720.png`

问题：

- 整体卡片在 1280x720 下偏高，底部恢复按钮贴近/超过首屏。
- 右侧空白较多，但内容区域反而拥挤。
- 安全感仍不足，危险操作和安全恢复操作的视觉分离还不够。

建议：

- 37D 中把恢复中心改成“备份列表 + 备份摘要 + 恢复操作区”。
- 危险操作“替换原项目”降级并单独放入危险区域。

### 设置

截图：

- `settings-small-laptop-1280x720.png`
- `settings-laptop-1366x768.png`

问题：

- 设置页是当前小屏问题最明显的模块之一。
- 双栏表单在 1280x720 下过挤，右侧配置组编辑器底部按钮被截。
- 配置组编辑器默认展开，使页面一上来就显得复杂。

建议：

- 37D 应优先重排设置页。
- API 配置组编辑器默认收起，点击“新建 / 编辑”后再展开。
- 配置组卡片显示状态摘要，避免直接展示完整编辑表单。
- 底部保存 / 测试类动作改成就近固定操作区或分组内动作。

## 优先级建议

1. 立即修小屏硬问题：书库左侧高度、设置页按钮裁切、Workshop 未打开项目仍显示输入区。
2. 继续 37C，但要把 Workshop 的无项目 / 无对话状态作为重点。
3. 37D 提前排上，设置页和恢复中心在小屏上比预期更需要处理。
4. 37B 不建议直接标为“视觉完全完成”，应标为“第一轮功能结构通过，笔记本视觉需 follow-up”。
